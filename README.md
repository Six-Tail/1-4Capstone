## 📝 **개발 과정**

### 1. **기획 및 디자인 단계**
- **UI/UX 디자인** (이태겸, 노종현)
    - 프로젝트의 초기 기획 단계에서, Figma를 사용하여 전체 애플리케이션의 디자인을 구상하고 사용자 경험(UX)과 인터페이스(UI)를 설계.
  - 주요 화면의 레이아웃을 구성하고, 사용자의 흐름을 고려하여 직관적이고 아름다운 디자인을 완성.


- **프로젝트 목표 및 스펙 정리** (배지호)
    - 프로젝트 매니저로서 애플리케이션의 주요 기능과 요구사항을 정리하고 팀원들과 기능 정의 회의를 통해 전체적인 개발 방향을 설정.

### 2. **프론트엔드 및 백엔드 개발**
- **화면 구현 및 UI/UX 반영** (이태겸, 조희섭, 배지호)
    - 플러터를 사용하여 애플리케이션의 주요 화면(홈 화면, 캘린더 화면, 게시판 화면 등)을 구현.
    - 디자인에 맞춰 UI 요소들을 배치하고, 사용자 상호작용(버튼 클릭, 드래그 등)을 처리하는 코드 작성.


- **백엔드 연동 및 데이터베이스 설계** (노종현, 배지호, 조희섭)
    - Firebase를 사용하여 데이터베이스를 설정하고, 실시간 데이터 동기화 및 서버 관리 작업을 수행.
    - 애플리케이션에서 필요한 API를 설계하고, 프론트엔드와의 연동을 통해 사용자 데이터를 처리할 수 있도록 구현.

### 3. **핵심 기능 개발**
- **캘린더 기능 개발** (배지호)
    - 사용자가 일정을 추가하고 수정할 수 있는 기능을 개발하고, 일정 삭제 및 날짜 선택 기능도 구현.
    - Firebase 데이터베이스와의 연동을 통해 일정 데이터를 실시간으로 저장하고 동기화.


- **커뮤니티 및 게시판 기능 개발** (조희섭)
    - 게시판 기능을 구현하여 사용자들이 게시물을 작성하고 관리할 수 있도록 하고, 댓글 기능 및 게시물의 수정/삭제 기능을 개발. 


- **랭킹 및 미션 기능 개발** (배지호)
    - 사용자가 목표를 달성하고 레벨을 올릴 수 있는 시스템을 구축.
    - 미션을 수행하며 레벨과 랭킹을 올리는 시스템을 설계하고, 사용자 성취도와 경험을 기록하여 동기화.


- **더보기 기능 개발** (조희섭, 정세운)
    - 추가 설정과 정보를 제공할 수 있는 '더보기' 기능을 개발하고, 이를 위한 UI와 동작을 구현.

### 4. **테스트 및 최종 준비**
- **버그 테스트 및 문제 해결** (정세운, 배지호, 조희섭)
    - 애플리케이션의 전반적인 기능을 테스트하고, 발견된 버그를 수정.
    - 각 기능별로 최적화와 오류 수정을 진행하며, 발표 전 마지막 확인 작업을 담당.


- **문서 작성** (노종현)
    - 프로젝트에서 사용된 기술과 구현된 API에 대한 문서를 작성하고, 코드와 기능을 설명하는 기술 문서를 정리.
    - 팀원 간의 협업을 원활하게 하기 위해 각종 기술 문서와 프로젝트 보고서를 작성하여 관리.
